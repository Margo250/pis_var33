# Use-case: Создание объявления в группе

**Первичный актор**: Администратор группы

**Цель**: Опубликовать важное объявление, которое увидят все участники группы.

**Предусловия**:
- Администратор авторизован в системе.
- Администратор является администратором хотя бы одной группы.
- Группа существует и активна.

**Основной поток** (Happy Path):
1. Администратор открывает раздел "Объявления" выбранной группы.
2. Система отображает форму для создания объявления.
3. Администратор вводит заголовок и текст объявления.
4. Администратор прикрепляет файл (например, документ или изображение) по желанию.
5. Администратор нажимает кнопку "Опубликовать".
6. Система валидирует данные.
7. Система сохраняет объявление в базе данных со статусом "Опубликовано".
8. Система отправляет уведомления всем участникам группы (асинхронно).
9. Система отображает администратору сообщение об успешной публикации.
10. Use-case завершен.

**Постусловия**:
- Объявление сохранено и доступно для просмотра участниками группы.
- В очереди задач находится задание на отправку уведомлений.

**Альтернативные потоки**:
- *3a. Администратор хочет запланировать публикацию*
  1. Администратор выбирает опцию "Запланировать".
  2. Администратор устанавливает дату и время публикации.
  3. Возврат к шагу 5. Система сохраняет объявление со статусом "Запланировано".
- *5a. Администратор нажимает "Сохранить как черновик"*
  1. Система сохраняет введенные данные со статусом "Черновик".
  2. Use-case завершается (уведомления не отправляются).

**Исключительные ситуации**:
- *6a. Валидация данных не пройдена (пустой заголовок, слишком длинный текст)*
  1. Система отображает сообщение об ошибке валидации.
  2. Система подсвечивает поля с ошибками.
  3. Возврат к шагу 3 для исправления.
- *7a. Ошибка при сохранении в базу данных (нарушение целостности, таймаут)*
  1. Система откатывает транзакцию.
  2. Система логирует ошибку.
  3. Система отображает сообщение: "Не удалось опубликовать объявление из-за технической ошибки. Попробуйте позже."
  4. Use-case завершается неудачей.
- *8a. Сервис уведомлений недоступен*
  1. Система успешно сохраняет объявление в БД.
  2. Система фиксирует ошибку отправки в логах.
  3. Система помещает задачу на отправку уведомлений в очередь для повторной попытки (Retry).
  4. Администратор видит сообщение: "Объявление опубликовано. Уведомления будут отправлены, как только сервис станет доступен."
  5. Use-case завершается успешно.
