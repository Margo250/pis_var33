## Ход выполнения работы

### 1. Команды (Commands)

**Созданные команды:**

1. **CreateAnnouncementCommand** - создание нового объявления
   - Поля: `group_id`, `author_id`, `title`, `content`, `attachment_ids`
   - Валидация: все поля не пустые, title и content не пустые строки
   - Файл: `application/command/create_announcement_command.py`

2. **PublishAnnouncementCommand** - публикация объявления
   - Поля: `announcement_id`
   - Валидация: announcement_id не пустой
   - Файл: `application/command/publish_announcement_command.py`

3. **ScheduleAnnouncementCommand** - планирование публикации
   - Поля: `announcement_id`, `scheduled_for`
   - Валидация: scheduled_for в будущем
   - Файл: `application/command/schedule_announcement_command.py`

4. **AddAttachmentCommand** - добавление вложения
   - Поля: `announcement_id`, `file_id`, `filename`, `file_size`, `mime_type`, `storage_path`
   - Валидация: file_size > 0 и <= 10MB
   - Файл: `application/command/add_attachment_command.py`

5. **RemoveAttachmentCommand** - удаление вложения
   - Поля: `announcement_id`, `file_id`
   - Файл: `application/command/remove_attachment_command.py`

6. **ArchiveAnnouncementCommand** - архивация объявления
   - Поля: `announcement_id`
   - Файл: `application/command/archive_announcement_command.py`

7. **UpdateAnnouncementContentCommand** - обновление содержимого
   - Поля: `announcement_id`, `title`, `content`
   - Валидация: title и content не пустые
   - Файл: `application/command/update_announcement_content_command.py`

**Пример кода команды:**
```python
@dataclass(frozen=True)
class CreateAnnouncementCommand:
    """Команда: создание нового объявления"""
    
    group_id: str
    author_id: str
    title: str
    content: str
    attachment_ids: Optional[List[str]] = None
    
    def __post_init__(self):
        if not self.group_id or not self.group_id.strip():
            raise ValueError("group_id не может быть пустым")
        
        if not self.author_id or not self.author_id.strip():
            raise ValueError("author_id не может быть пустым")
        
        if not self.title or not self.title.strip():
            raise ValueError("title не может быть пустым")
        
        if not self.content or not self.content.strip():
            raise ValueError("content не может быть пустым")
```

### 2. Command Handlers

**Созданные обработчики:**

1. **CreateAnnouncementHandler** - создание объявления
   - Шаги обработки: генерация ID, создание доменного объекта Announcement, сохранение в репозиторий
   - Возвращает: ID созданного объявления
   - Файл: `application/command/handlers/create_announcement_handler.py`

2. **PublishAnnouncementHandler** - публикация объявления
   - Шаги: загрузка Announcement, вызов publish(), сохранение, отправка уведомлений
   - Файл: `application/command/handlers/publish_announcement_handler.py`

3. **ScheduleAnnouncementHandler** - планирование публикации
   - Шаги: загрузка Announcement, вызов schedule()
   - Файл: `application/command/handlers/schedule_announcement_handler.py`

4. **AddAttachmentHandler** - добавление вложения
   - Шаги: загрузка Announcement, создание FileAttachment, вызов add_attachment()
   - Файл: `application/command/handlers/add_attachment_handler.py`

5. **RemoveAttachmentHandler** - удаление вложения
   - Шаги: загрузка Announcement, вызов remove_attachment()
   - Файл: `application/command/handlers/remove_attachment_handler.py`

6. **ArchiveAnnouncementHandler** - архивация объявления
   - Шаги: загрузка Announcement, вызов archive()
   - Файл: `application/command/handlers/archive_announcement_handler.py`

7. UpdateAnnouncementContentHandler - обновление содержимого
   - Шаги: загрузка Announcement, вызов update_content()
   - Файл: `application/command/handlers/update_announcement_content_handler.py`

**Пример кода handler:**
```python
class PublishAnnouncementHandler:
    """Обработчик команды публикации объявления"""
    
    def __init__(
        self,
        repository: AnnouncementRepository,
        notification_service: NotificationService
    ):
        self._repository = repository
        self._notification_service = notification_service
    
    def handle(self, command: PublishAnnouncementCommand) -> None:
        announcement = self._repository.find_by_id(command.announcement_id)
        
        if not announcement:
            raise ValueError(f"Announcement not found: {command.announcement_id}")
        
        announcement.publish()
        self._repository.save(announcement)
        
        self._notification_service.send_announcement_published(
            announcement_id=announcement.id,
            group_id=announcement.group_id
        )
```

### 4. Query Handlers

**Созданные обработчики запросов:**

1. **GetAnnouncementByIdHandler** - получение объявления по ID
   - Репозиторий: `AnnouncementRepository`
   - Возвращает: `AnnouncementDTO`
   - Файл: `application/query/handlers/get_announcement_by_id_handler.py`

2. **ListAnnouncementsByGroupHandler** - список объявлений в группе
   - Репозиторий: `AnnouncementRepository`
   - Возвращает: `List[AnnouncementDTO]`
   - Файл: `application/query/handlers/list_announcements_by_group_handler.py`

3. **ListAnnouncementsByAuthorHandler** - список объявлений автора
   - Репозиторий: `AnnouncementRepository`
   - Возвращает: `List[AnnouncementDTO]`
   - Файл: `application/query/handlers/list_announcements_by_author_handler.py`

**Пример кода:**
```python
class GetAnnouncementByIdHandler:
    """Обработчик запроса получения объявления по ID"""
    
    def __init__(self, repository: AnnouncementRepository):
        self._repository = repository
    
    def handle(self, query: GetAnnouncementByIdQuery) -> AnnouncementDTO:
        announcement = self._repository.find_by_id(query.announcement_id)
        
        if not announcement:
            raise ValueError(f"Announcement not found: {query.announcement_id}")
        
        return self._to_dto(announcement)
    
    def _to_dto(self, announcement: Announcement) -> AnnouncementDTO:
        attachments = [
            AttachmentDTO(
                file_id=a.file_id,
                filename=a.filename,
                file_size=a.file_size,
                mime_type=a.mime_type
            )
            for a in announcement.attachments
        ]
        
        return AnnouncementDTO(
            id=announcement.id,
            group_id=announcement.group_id,
            author_id=announcement.author_id,
            title=announcement.title,
            content=announcement.content,
            status=announcement.status.value,
            attachments=attachments,
            created_at=announcement.created_at,
            published_at=announcement.published_at,
            scheduled_for=announcement.scheduled_for,
            updated_at=announcement.updated_at
        )
```

### 5. Application Service (Фасад)

**Реализованный сервис:** `AnnouncementServiceImpl`

**Методы:**

| Метод | Тип | Возвращает |
|-------|-----|------------|
| create_announcement(command) |	Command	| str (ID) |
| publish_announcement(command) |	Command |	void |
| schedule_announcement(command) |	Command |	void |
| add_attachment(command) |	Command |	void |
| remove_attachment(command) |	Command |	void |
| archive_announcement(command) |	Command	| void |
| update_announcement_content(command) |	Command	| void |
| get_announcement_by_id(query) |	Query |	AnnouncementDTO |
| list_announcements_by_group(query) |	Query |	List[AnnouncementDTO] |
| list_announcements_by_author(query)	| Query |	List[AnnouncementDTO] |

**Пример кода:**
```python
class AnnouncementServiceImpl(AnnouncementService):
    """Реализация фасада прикладного слоя"""
    
    def __init__(
        self,
        create_handler: CreateAnnouncementHandler,
        publish_handler: PublishAnnouncementHandler,
        schedule_handler: ScheduleAnnouncementHandler,
        add_attachment_handler: AddAttachmentHandler,
        remove_attachment_handler: RemoveAttachmentHandler,
        archive_handler: ArchiveAnnouncementHandler,
        update_content_handler: UpdateAnnouncementContentHandler,
        get_by_id_handler: GetAnnouncementByIdHandler,
        list_by_group_handler: ListAnnouncementsByGroupHandler,
        list_by_author_handler: ListAnnouncementsByAuthorHandler
    ):
        self._create_handler = create_handler
        self._publish_handler = publish_handler
        self._schedule_handler = schedule_handler
        self._add_attachment_handler = add_attachment_handler
        self._remove_attachment_handler = remove_attachment_handler
        self._archive_handler = archive_handler
        self._update_content_handler = update_content_handler
        self._get_by_id_handler = get_by_id_handler
        self._list_by_group_handler = list_by_group_handler
        self._list_by_author_handler = list_by_author_handler
    
    def create_announcement(self, command: CreateAnnouncementCommand) -> str:
        return self._create_handler.handle(command)
    
    def get_announcement_by_id(self, query: GetAnnouncementByIdQuery) -> AnnouncementDTO:
        return self._get_by_id_handler.handle(query)
```

## Вывод

В ходе выполнения лабораторной работы реализован прикладной слой для сервиса объявлений с разделением на команды и запросы (CQRS на уровне интерфейса).
**Что реализовано**:
- 7 команд (Create, Publish, Schedule, AddAttachment, RemoveAttachment, Archive, UpdateContent) с иммутабельными DTO и валидацией
- 7 Command Handlers, реализующих оркестрацию вызовов домена
- 3 запроса (GetById, ListByGroup, ListByAuthor) с Read DTO
- 3 Query Handlers для преобразования доменных моделей в DTO
- Application Service (фасад), делегирующий вызовы специализированным handlers
- Юнит-тесты для проверки работы handlers и валидации
**Проблемы**:
- Формат даты в команде ScheduleAnnouncementCommand требует дополнительной валидации на уровне приложения
- В Query Handler для списка по автору потребовалось добавить метод в репозиторий

Разделение команд и запросов чёткое — команды изменяют состояние, запросы только читают. Все обработчики покрыты юнит-тестами. Application Service выступает единой точкой входа для внешних адаптеров.
