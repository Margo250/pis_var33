#### Исходящий адаптер: Repository

**InMemoryAnnouncementRepository**:
```python
class InMemoryAnnouncementRepository(AnnouncementRepository):
    def __init__(self):
        self._storage = {}
    
    def save(self, announcement) -> None:
        self._storage[announcement.id] = announcement
    
    def find_by_id(self, announcement_id: str):
        return self._storage.get(announcement_id)
```

### Часть 6. Dependency Injection (Конфигурация зависимостей)

**DependencyContainer**:
```python
class DependencyContainer:
    def __init__(self):
        self.announcement_repository = InMemoryAnnouncementRepository()
        self.notification_service = ConsoleNotificationService()
        
        self.announcement_service = AnnouncementService(
            repository=self.announcement_repository,
            notification_service=self.notification_service
        )
    
    def get_announcement_service(self) -> AnnouncementService:
        return self.announcement_service
```

**Как работает DI**: Контейнер создаёт экземпляры исходящих адаптеров (репозиторий, нотификатор) и передаёт их в конструктор AnnouncementService. Сервис сам не создаёт зависимости, получает их извне.

## 6. Выводы

### Что получилось хорошо
Удалось чётко отделить бизнес-логику (Announcement, Group, User) от инфраструктуры. Domain слой не имеет зависимостей от фреймворков или БД. AnnouncementService легко тестируется с моками.

### С какими трудностями столкнулись
Было сложно понять, зачем создавать интерфейс для порта, если у него только одна реализация. После изучения принципа Dependency Inversion стало ясно: это для тестируемости и возможности замены адаптера (например, InMemory на PostgreSQL).

### Что узнали нового
Изучил гексагональную архитектуру (Ports & Adapters), принцип Dependency Inversion, Dependency Injection. Понял, как изоляция слоёв упрощает замену БД или UI без изменения бизнес-логики.

### Как можно улучшить
Добавить реальную БД (PostgreSQL), реализовать EventBus для асинхронных уведомлений, написать юнит-тесты для AnnouncementService.
